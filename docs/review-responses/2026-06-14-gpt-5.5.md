## Deferral rationale

- **Sweep blocks the event loop.** Kept on purpose. Moving it to a worker thread
  reintroduces the MPS thread-safety segfault we fixed earlier (PyTorch MPS must
  be driven from one thread). For a local single-user app a few seconds of paused
  stream is acceptable; the UI now states it. Revisit only if we host this.
- **O(N²) communication.** Capped, not redesigned. Proper spatial binning is the
  real fix and lands when **Lesson 0.6 (nutrient cycling)** scales populations up.
- **`make_policy` fallback / `survival_food_reward` masking.** Both are in code
  paths that only the upcoming RL lessons exercise; finalized there.
